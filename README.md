# Agent 全局 Skills 说明

这个目录存放当前用户自定义的 Agent 全局技能。每个技能使用独立目录管理，核心入口文件为 `SKILL.md`。

## 目录用途

- 作用范围：放在 `~/.agents/skills/` 下的技能，可作为 Agent 全局技能被发现和使用。
- 适用场景：沉淀可复用的系统排查、技能发现、Git 提交消息生成和 Git 提交流程。
- 组织方式：一个技能一个目录；有补充模板、参考文档或脚本时，放在技能子目录内统一维护。

## 当前技能

以下列表只包含当前仓库中带有 `SKILL.md` 的用户自定义技能目录；`.git/`、`.idea/` 和未提供技能入口的辅助目录不计入“当前技能”。

| 技能目录 | 主要用途 |
| --- | --- |
| `disk-usage-report` | 分析当前电脑磁盘使用情况，统计整体容量、占用最高目录、内容类型和适合清理的目标，帮助安全释放空间。 |
| `find-skills` | 帮助发现和安装开放 Agent skills 生态中的技能，适用于查找特定领域能力、工具、模板或工作流。 |
| `git-commit-msg` | 根据用户确认的 Git 改动范围生成规范化提交消息；只输出提交消息，不执行 `git add`、`git commit` 或 `git push`。 |
| `git-push` | 推送 Git 提交：先确认提交文件范围，再生成并确认提交信息，随后按确认范围执行 `git add`、`git commit`，最后在用户明确同意时执行 `git push`。 |
| `operation-docs` | 为单个中间件或服务生成/更新运维文档，沉淀安装位置、启停方式、配置、日志、依赖影响和常见运维操作。 |
| `process-usage-report` | 分析当前电脑进程使用情况，统计进程数量、CPU/内存/IO 高占用进程和分类分布，帮助定位卡顿原因。 |
| `server-operation-index` | 生成服务器全局运维总览文档，汇总已部署服务、子文档索引、端口矩阵、依赖关系和维护记录。 |

## 使用方式

1. 在会话中直接描述需求，命中技能适用场景时，Agent 会按技能说明执行。
2. 如果想明确触发某个技能，可以在需求中直接点名技能名称或表达对应任务目标。
3. 使用前优先检查对应目录下的 `SKILL.md`，必要时补充 `references/`、模板或脚本资源。
4. 新增技能时，至少创建 `技能目录/SKILL.md`，写清名称、用途、触发条件、执行步骤和边界约束。
5. 涉及用户确认或选择的 Git 流程，必须遵守对应 `SKILL.md` 中的交互约束，不得跳过确认节点。
6. 系统排查类技能默认只做采集、分析和建议，不直接删除文件、结束进程或执行不可逆操作。
7. 运维文档类技能若新增或移除服务子文档，应同步检查总览索引技能与相关模板是否仍匹配当前流程。

## 编写建议

- 描述要聚焦“什么时候用、具体怎么做、哪些不能做”。
- 尽量把输出路径、交付物格式、确认节点和安全边界写清楚，减少执行歧义。
- 有固定模板或参考材料时，放进技能目录并在 `SKILL.md` 中显式引用。
- 技能面向复用，优先沉淀稳定流程，不把一次性上下文写进全局技能。
- 技能触发方式、关键资源、维护约束发生变化时，优先更新对应 `SKILL.md`，并同步检查本 README 的“当前技能”和使用说明是否仍准确。

## 备注

- 本 README 只介绍 `~/.agents/skills/` 下带有 `SKILL.md` 的用户自定义技能。
- 技能内容调整后，建议同步更新对应目录的 `SKILL.md`，保持 README 与实际能力一致。
